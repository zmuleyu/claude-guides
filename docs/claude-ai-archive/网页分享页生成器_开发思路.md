# 网页分享页生成器 — 开发思路

> 在 Chrome 滚动截屏扩展基础上，新增"将当前浏览网页生成分享页"能力，输出符合小红书图片轮播和微信公众号图文格式。

---

## 一、基座选择

| 候选方案 | 适配度 | 理由 |
|---------|--------|------|
| **Chrome 滚动截屏扩展** | ⭐⭐⭐ | 已有 content script 注入、Canvas 拼接、OffscreenDocument、多格式导出管线，天然可扩展 |
| Claude 插件 | ⭐ | 定位是 claude.ai 任务自动化，与通用网页内容提取无关 |
| Tampermonkey 脚本 | ⭐⭐ | 能做内容提取，但缺 offscreen Canvas 能力和 `chrome.downloads` API |

**结论**：以 `Chrome滚动截屏扩展_开发方案.md` 为基座，新增 `share-page-generator` 模块。

## 二、目标平台格式约束

### 2.1 小红书（图片轮播 1~9 张）

| 维度 | 要求 |
|------|------|
| 载体 | 多张图片（1~9 张），不支持富文本正文 |
| 推荐尺寸 | 3:4（1080×1440px），也接受 1:1（1080×1080px） |
| 封面图 | 大标题 + 视觉冲击 + 话题关键词 |
| 正文图 | 文字卡片 / 信息图，每张聚焦一个要点 |
| 尾图 | CTA / 关注引导 / 来源说明 |
| 视觉风格 | 高对比背景、圆角卡片、大标题+小正文、emoji 点缀 |
| 输出格式 | 一组 PNG/JPEG 文件（或 ZIP 打包） |

### 2.2 微信公众号（富文本 HTML）

| 维度 | 要求 |
|------|------|
| 载体 | HTML 富文本，通过公众号编辑器粘贴 |
| 正文区宽度 | 固定 578px |
| CSS 限制 | 不支持外链 CSS/JS，只支持 inline style |
| 图片限制 | 需 base64 或已上传到微信 CDN 的 URL，外链图片不显示 |
| 常见排版 | 引导关注头图 → 正文段落（16px/行高2）→ 图片穿插 → 尾部引导 |
| 输出格式 | 可直接粘贴到公众号编辑器的 HTML 片段 |

## 三、核心管线架构

```
[当前网页]
  → ① Content Extractor （提取结构化内容）
  → ② Template Engine    （套用平台模板）
  → ③ Renderer           （渲染为目标格式）
  → ④ Exporter           （导出 / 下载 / 剪贴板）
```

### 3.1 Content Extractor（content script 中运行）

**输出数据结构**：

```typescript
interface ExtractedContent {
  title: string;
  author?: string;
  publishDate?: string;
  heroImage?: string;              // 首图 URL 或 base64
  sections: ContentSection[];      // 正文分段
  metadata: Record<string, string>; // og:tags / JSON-LD 等
}

interface ContentSection {
  type: 'heading' | 'paragraph' | 'image' | 'code' | 'quote' | 'list';
  content: string;                 // 文本内容或图片 URL
  level?: number;                  // heading 层级
}
```

**提取策略优先级**：

1. **Readability 算法**（参考 Mozilla Readability.js）— 适合文章类页面，自动剥离导航/广告/侧边栏
2. **结构化数据**（JSON-LD、Open Graph meta）— 补充标题/作者/首图等元信息
3. **选区提取**（P1）— 用户手动框选页面区域，只提取选中部分
4. **全页提取**（fallback）— 提取 `<body>` 中所有可见文本和图片

### 3.2 Template Engine

将 `ExtractedContent` 映射到平台特定的视觉模板。

**小红书模板系统**：

- 每张图 = 一个 Canvas 渲染任务
- 内置 3~5 套风格预设（简约白 / 科技蓝 / 文艺米 / 商务灰 / 暗黑）
- 模板定义项：背景色/渐变、字体族/大小/颜色、布局区域划分、装饰元素（线条/图标/emoji）
- **自动分页算法**：根据文字量 + 图片数量计算需要几张卡片，每张卡片容纳约 80~120 字 + 可选配图

**微信模板系统**：

- 模板 = 一组 inline CSS 规则 + HTML 骨架结构
- 内置几套常见公众号排版风格（极简 / 杂志 / 技术博客）
- 图片处理管线：外链图片 URL → service-worker `fetch` → Canvas → base64 内联（绕过公众号外链限制）

### 3.3 Renderer

**小红书路径**（Canvas 渲染）：

```
Template + Content
  → OffscreenDocument 中逐张 Canvas 渲染
  → canvas.toBlob('image/png')
  → Blob[]
```

复用现有截屏扩展的 offscreen 架构。当前 `offscreen.ts` 已有 Canvas 拼接能力，扩展为"模板渲染"：

```
现有能力：截图帧[] → Canvas 拼接 → 单张长图
新增能力：ExtractedContent + Template → Canvas 逐张渲染 → 多张卡片图
```

**微信路径**（HTML 拼接）：

```
Template + Content
  → 纯字符串拼接（inline style HTML）
  → 不需要 Canvas
  → 在 service-worker 或 content script 中完成
```

### 3.4 Exporter

| 平台 | 输出形式 | 实现方式 |
|------|---------|---------|
| 小红书 | 多张 PNG 逐一下载 | `chrome.downloads.download()` |
| 小红书 | ZIP 打包下载 | JSZip (~100KB) 打包后下载 |
| 小红书 | 预览轮播 | popup / side panel 中缩略图轮播 |
| 微信 | 剪贴板富文本 | `navigator.clipboard.write(ClipboardItem)` 写入 HTML |
| 微信 | .html 文件下载 | `chrome.downloads.download()` |
| 通用 | 单张长图 | 复用现有截屏管线 |

## 五、消息协议扩展

在现有 `Message.type` 基础上新增：

```typescript
// 新增消息类型
| 'EXTRACT_CONTENT'       // popup → content: 提取当前页面内容
| 'CONTENT_EXTRACTED'     // content → sw: 提取完成，携带 ExtractedContent
| 'RENDER_CARDS'          // sw → offscreen: 渲染小红书卡片图
| 'CARDS_RENDERED'        // offscreen → sw: 卡片渲染完成，携带 Blob[]
| 'RENDER_WECHAT_HTML'    // sw 内部: 渲染微信 HTML（不需要 offscreen）
| 'COPY_TO_CLIPBOARD'     // sw → content: 将 HTML 写入剪贴板
```

## 六、关键技术风险与缓解

| # | 风险 | 影响 | 缓解措施 |
|---|------|------|---------|
| 1 | **Canvas 中文字体缺失** | OffscreenDocument 默认无中文字体，渲染出方块 | 通过 `FontFace` API 加载打包的 woff2 字体；选用 Noto Sans SC（压缩后约 2~4MB） |
| 2 | **图片跨域 CORS** | 网页图片 `drawImage` 后调用 `toBlob` 报 tainted canvas | 在 service-worker 中 `fetch` 图片 → 转 base64 → 传入 offscreen Canvas |
| 3 | **微信编辑器 CSS 白名单** | 部分 CSS 属性被公众号编辑器剥离 | 实测确认支持的 style 子集；仅使用安全属性（color/font-size/line-height/margin/padding/text-align/background-color） |
| 4 | **Readability 提取质量** | SPA 页面、动态加载、iframe 嵌套场景下失败 | 提供手动选区模式作为 fallback；检测 SPA 路由后等待 DOM 稳定 |
| 5 | **扩展体积膨胀** | 中文字体 woff2 增加 2~4MB 包体积 | 按需加载（首次使用时下载到 IndexedDB 缓存）；或使用字体子集化工具裁剪字符集 |
| 6 | **小红书图片上传限制** | 生成的图片可能被平台压缩/拒绝 | 输出标准 sRGB PNG/JPEG；控制单张文件 < 5MB；尺寸严格 1080×1440 |

## 七、分阶段开发路线

### P0 — 最小可用版本（第 1~3 周）

| 任务 | 交付物 | 验收标准 |
|------|--------|---------|
| Content Extractor 基础版 | `content-extractor.ts` | 能从文章类页面提取标题/正文/首图，覆盖 10+ 主流网站 |
| 小红书单模板渲染 | offscreen Canvas 渲染管线 + 1 套模板 | 输入 ExtractedContent → 输出 3:4 PNG 卡片图组 |
| PNG 多图导出 | exporter 扩展 | 点击按钮 → 自动下载全部卡片图 |
| Popup UI 入口 | popup 新增 "分享页" tab | 选择平台 → 预览缩略图 → 一键导出 |

### P1 — 双平台 + 多模板（第 4~6 周）

| 任务 | 交付物 | 验收标准 |
|------|--------|---------|
| 微信 HTML 模板渲染 | `template-engine.ts` + wechat 模板 | 生成的 HTML 可直接粘贴到公众号编辑器并正确显示 |
| 剪贴板富文本写入 | exporter 扩展 | 一键复制 → 在公众号编辑器 Ctrl+V 即呈现排版 |
| 多模板选择 UI | popup/side panel | 提供 3~5 套风格预览，点击切换实时更新 |
| 图片 base64 内联管线 | service-worker fetch 代理 | 微信模板中的图片全部内联为 base64 |
| 中文字体加载 | FontFace API + woff2 | Canvas 渲染正确显示中文 |

### P2 — 体验增强（第 7~9 周）

| 任务 | 交付物 | 验收标准 |
|------|--------|---------|
| 用户选区提取模式 | 复用 region-selector 或新建 text-selector | 框选页面区域 → 只提取选中内容生成分享页 |
| ZIP 打包下载 | JSZip 集成 | 小红书卡片一次性打包为 `share-cards.zip` |
| 自定义参数面板 | popup UI 扩展 | 可调：标题文案、配色方案、字体大小、每张卡片最大字数 |
| 实时预览 | popup/side panel 内嵌预览 | 参数修改后即时刷新预览图 |

### P3 — 智能化（第 10~12 周）

| 任务 | 交付物 | 风险 |
|------|--------|------|
| Claude API 辅助内容摘要 | API 调用模块 | 需要 API Key 配置，增加用户使用门槛 |
| 智能排版建议 | 根据内容类型自动推荐模板 | 准确度有限 |
| 自定义模板编辑器 | 可视化模板配置 UI | 开发量大，早期 ROI 低 |
| 多语言字体支持 | 日/韩/繁体字体包 | 进一步增加包体积 |

## 八、小红书卡片自动分页算法

核心逻辑（`card-paginator.ts`）：

```
输入：ExtractedContent + TemplateConfig
输出：CardPage[]（每个 CardPage 包含该页要渲染的内容片段）

步骤：
1. 第 1 页固定为封面：标题 + heroImage + 来源/作者
2. 计算每页可容纳的文字量：
   - 页面尺寸 1080×1440
   - 减去上下 padding（各 120px）、标题区域（200px）
   - 正文区域高度 ≈ 1000px
   - 按 font-size 32px + line-height 1.8 → 每行约 17 字，约 17 行
   - 每页 ≈ 250~300 字（留白后）
3. 遍历 sections，按字数阈值分页
4. 遇到 image 类型的 section → 该图片独占一页或与前文同页（取决于剩余空间）
5. 最后一页为尾图：来源 URL + 关注引导 + 日期
6. 总页数 > 9 时：压缩正文密度或提示用户精简内容
```

## 九、微信 HTML 安全 CSS 子集

基于实测，公众号编辑器可靠支持的 inline style 属性：

```
/* 文本 */
color, font-size, font-weight, font-style, font-family,
line-height, letter-spacing, text-align, text-decoration, text-indent

/* 盒模型 */
margin, margin-top/right/bottom/left,
padding, padding-top/right/bottom/left,
width, max-width, min-height

/* 背景与边框 */
background-color, background-image (linear-gradient),
border, border-radius, border-bottom, box-shadow (部分)

/* 布局 */
display (block/inline-block/flex 部分),
overflow: hidden

/* 不可靠 / 不支持 */
position, float, z-index, transform, animation,
外链 background-image url(), @media, @font-face
```

**原则**：只用以上安全属性；不用 class/id（会被剥离）；图片必须 base64 或微信 CDN URL。
