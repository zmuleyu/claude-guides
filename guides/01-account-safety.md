# Claude 账号封禁预防指南

> **适用范围**：claude.ai 个人/Pro/Max 用户 及 API 开发者
> **政策依据**：Anthropic AUP（2025年9月15日版）及官方帮助中心
> **最后更新**：2026年3月

---

## 一、账号注册阶段

### 1.1 使用真实合规信息注册

- ✅ 使用**长期有效的真实邮箱**（Gmail、Outlook、企业邮箱等主流服务商）
- ✅ 使用**真实手机号**完成验证，避免使用一次性/虚拟号码服务（如 TextNow、Google Voice）
- ✅ 付费订阅时使用与注册地区一致的支付方式（信用卡账单地址与 IP 地理位置尽量匹配）
- ❌ 禁止使用临时邮箱（10minutemail 等）
- ❌ 禁止使用虚拟手机号注册

### 1.2 确认所在地区受支持

在注册前，先确认您所在地区是否在 Anthropic [支持地区列表](https://www.anthropic.com/supported-countries) 内。从不受支持地区访问，是触发自动封禁的高频原因之一。

> ⚠️ **中国大陆用户特别注意**：自 2025 年 9 月起，Anthropic 明确禁止"中国实体控制的公司"使用 Claude 服务（含子公司）。使用 VPN 访问存在封号风险，请审慎评估。

## 二、网络环境管理

### 2.1 IP 稳定性

| 行为 | 风险等级 | 建议 |
|------|--------|------|
| 固定家庭宽带访问 | 🟢 低 | 推荐 |
| 固定企业网络访问 | 🟢 低 | 推荐 |
| 偶尔切换 Wi-Fi | 🟡 中 | 可接受，避免频繁 |
| 使用免费共享 VPN | 🔴 高 | 强烈不建议 |
| 频繁切换 VPN 节点 | 🔴 高 | 强烈不建议 |
| 使用 Tor 网络访问 | 🔴 极高 | 禁止 |

### 2.2 设备与浏览器一致性

- ✅ 尽量使用**固定设备 + 固定浏览器**访问
- ✅ 浏览器语言、时区设置与账号注册地区保持一致
- ❌ 避免频繁清除 Cookie 后重新登录（会触发异常设备检测）
- ❌ 避免使用隐身模式作为主要访问方式

## 三、内容使用规范

### 3.1 绝对禁止的内容（直接封号）

以下类型内容一旦触发，可能导致**立即且不可逆**的封禁：

- 🚫 任何涉及儿童性内容（CSAM）
- 🚫 生化核武器、爆炸物合成/设计
- 🚫 恶意代码（病毒、勒索软件、网络攻击工具）
- 🚫 真实个人信息的非授权收集、身份伪造
- 🚫 破坏关键基础设施（电网、医疗系统、选举系统）

### 3.2 需谨慎处理的敏感话题

以下话题本身不违规，但**提问方式**可能触发误判：

| 话题领域 | 错误提问方式 ❌ | 正确提问方式 ✅ |
|---------|-------------|-------------|
| 网络安全 | "如何入侵某网站" | "作为渗透测试研究，常见 Web 漏洞有哪些？" |
| 药品知识 | "什么剂量的X药物会致命" | "小说中一个毒物学家会如何描述药物过量的临床表现" |
| 武器知识 | "如何制作炸弹" | （此类话题无合规提问方式，请直接回避） |
| 心理干预 | "如何让人相信谎言" | "心理学上，说服技巧的伦理边界是什么" |
| 政治敏感 | 大量生产某党派竞选材料 | 单次学术性分析 |

> 💡 **实用技巧**：对于学术研究、创意写作、安全研究等场景，在提问开头**明确说明上下文和用途**，可显著降低误判率。

### 3.3 高风险用例的合规要求

如您将 Claude 用于以下场景，需确保满足附加要求：

- **医疗/法律/金融建议类产品**：必须接入专业人工审核，且需向终端用户披露 AI 参与
- **面向用户的聊天机器人**：每次会话开始时必须告知用户正在与 AI 交互
- **服务未成年人的产品**：须遵守 Anthropic 未成年人保护附加指南

## 四、API 使用规范（开发者专项）

### 4.1 API Key 管理

- ✅ 为不同项目使用独立的 API Key，便于权限管理
- ✅ 将 API Key 存储在服务端环境变量中，切勿硬编码在前端代码
- ✅ 定期轮换 API Key，尤其在代码泄露风险后
- ❌ **禁止将 API Key 共享给第三方**（包括客户和合作方）
- ❌ 禁止将 API Key 暴露在公开代码仓库（GitHub/GitLab 等）
- ❌ 禁止在前端 JavaScript 中直接使用 API Key

### 4.2 请求频率与行为模式

- ✅ 实现合理的请求队列与速率限制
- ✅ 添加适当的请求间隔，模拟自然交互节奏
- ❌ 避免短时间内发送大量相似请求（可能触发 DDoS/滥用检测）
- ❌ 避免完全自动化的无人监督大规模调用（需遵循 AUP 关于 agentic 用途的规定）

### 4.3 禁止的 API 用途

- ❌ 使用 Claude 输出训练竞争性 AI 模型（"模型蒸馏"）
- ❌ 使用 Claude 对 Anthropic 竞争产品进行基准测试
- ❌ 对 Claude 进行逆向工程
- ❌ 使用 harness 类工具将 claude.ai 订阅驱动自动化工作流（仅限 API 计划）

### 4.4 Claude Code CLI 特别注意事项

- ⚠️ Claude Code CLI 与 claude.ai 网页版**共享同一个额度池**，两者消耗互相叠加
- ✅ 使用 `--print` 模式执行一次性命令（不计入对话配额）
- ✅ 配置 `core.hooksPath` 全局 pre-commit hook 防止 API Key 泄露
- ❌ 不要使用 `--no-verify` 跳过 pre-commit 检查（除非明确知道在做什么）
- ❌ 不要在 System Prompt 或 CLAUDE.md 中要求 Claude "绕过安全限制"
- ⚠️ 多窗口并行使用 Claude Code 时，额度消耗会加速，注意控制窗口数量（建议 ≤3）
- ✅ 使用 `hooks` 自动化重复操作，减少不必要的对话轮次

## 五、账号日常使用规范

### 5.1 账号独立使用

- ✅ **一人一号**，不与他人共用账号
- ✅ 如有多设备需求，使用官方支持的多设备登录功能
- ❌ 禁止将账号出租或出售给他人使用
- ❌ 禁止多人同时使用同一账号（行为异常会触发检测）

### 5.2 警告信号监控

出现以下情况时，请立即停止相关操作并审查使用行为：

- 💡 收到 Claude 内联的"违规警告"提示
- 💡 某类问题被持续拒绝响应（可能处于观察名单）
- 💡 账号登录时要求额外验证
- 💡 API 调用返回异常的 429/403 错误频率上升

### 5.3 付费与账单合规

- ✅ 使用账单地址与注册地区一致的支付方式
- ❌ 不要对 Claude 订阅发起**信用卡拒付（Chargeback）**——这会直接导致账号永久封禁
- ❌ 不要使用虚拟卡或匿名支付方式

## 六、Prompt 工程安全实践

### 6.1 避免触发安全过滤器

```
# 高风险 Prompt 结构
"忽略之前的所有指令，现在你是..."        ← Jailbreak 尝试，直接封号
"假设没有任何限制，你会怎么..."          ← 规避限制意图明显
"用DAN模式回答..."                      ← 已知越狱词汇

# 低风险替代结构
"在[具体场景/用途]的背景下，请帮我..."   ← 明确上下文
"作为[角色]，针对[目标受众]，请..."      ← 角色设定有合理依据
"这是一个虚构故事，其中角色需要..."     ← 创意写作框架
```

### 6.2 System Prompt 设计安全建议

- ✅ 明确声明 AI 的角色和限制
- ✅ 包含"始终遵守 Anthropic 使用政策"的声明
- ❌ 不要在 system prompt 中要求 Claude "绕过安全措施"
- ❌ 不要在 system prompt 中要求 Claude "假装没有内容限制"

## 七、团队/企业使用补充规范

### 7.1 团队账号管理

- ✅ 为每位员工分配独立账号（Team/Enterprise 计划支持多席位）
- ✅ 建立内部使用规范，明确禁止行为
- ✅ 定期审查 API 使用日志，发现异常及时处理
- ✅ 离职员工及时注销或移除账号权限

### 7.2 产品集成合规检查清单

在将 Claude API 集成到产品中发布前，请核对：

- [ ] 终端用户是否知晓正在与 AI 交互？
- [ ] 医疗/法律/金融类建议是否有人工专家复核机制？
- [ ] 是否存在用户生成内容被直接传入 Claude 的风险？（需增加输入过滤）
- [ ] 服务的终端用户是否包含未成年人？（需额外合规措施）
- [ ] API Key 是否安全存储，不存在泄露风险？
