# Claude 使用效率方法论

> 从时间管理、Prompt 工程、工具配置三个维度最大化 Claude 使用效率。
> 适用于所有 Claude Pro/Max 用户和 Claude Code 用户。

---

## 一、理解额度机制

### 5 小时滑动窗口

```
⸰ 不是固定的"每天重置"
⸰ 从你发送第一条消息时开始计时
⸰ 5小时后额度刷新，下一个窗口开始

示例：
10:00 发第1条消息 → 窗口A开始
10:00–15:00 = 窗口A
15:00 窗口A到期 → 可开始窗口B
```

### 适用范围

| 平台 | 是否受额度限制 |
|------|---------------|
| Claude.ai 网页版 | ✅ |
| Claude 桌面客户端 | ✅ |
| Claude 移动端 | ✅ |
| Claude Code（CLI） | ✅ |
| API（按 token 计费） | ❌ 不适用 |

> ⚠️ Claude Code 与 Claude Web 共用同一个额度池，两者消耗会互相叠加。

## 二、时间窗口优化

### 预热占位技巧

在正式工作开始前，提前发一条简单消息启动窗口计时器：

```bash
# Claude Code 用户
alias claude-ping='echo "hi" | claude --print'

# 使用时：
claude-ping  # 启动窗口计时
# ... 正常工作开始前有"空闲"的额度消耗时间
```

**窗口衔接计算**：
```
公式：占位时间 = 正式开工时间 - (5小时 - 预计耗尽时长)

示例：
- 10:00 开始正式写代码
- 通常 2 小时用完额度
- 占位时间 = 10:00 - (5 - 2)h = 07:00
- 07:00 发一条 "hi" → 窗口到 12:00 到期
- 10:00–12:00 正式使用（2h）
- 12:00 第二窗口自动开始，新的额度
```

## 三、高密度 Prompt 工程

### 核心原则

**减少轮次，最大化每条消息的信息密度。**

### 高效结构模板

```markdown
# 任务打包请求

## 背景与上下文
[一次性提供所有必要背景，不要分多条补充]

## 任务列表（请按顺序执行）
1. [具体任务1，含成功标准]
2. [具体任务2，含成功标准]
3. [具体任务3，含成功标准]

## 约束条件
- [约束1]
- [约束2]

## 输出格式要求
- 任务1：[期望格式]
- 任务2：[期望格式]
- 任务3：[期望格式]

## 代码/材料
[在此粘贴所有相关代码、文档或数据]
```

### 反面 vs 正面示例

❌ **低效用法**（消耗 4 次额度）：
```
第1条：帮我分析这段代码有什么问题
第2条：好的，那帮我修复这些问题
第3条：能给这段代码写测试吗
第4条：帮我写一下函数注释
```

✅ **高效用法**（消耗 1 次额度）：
```
请对以下代码完成4件事：
1. 分析潜在问题（安全/性能/可维护性各维度）
2. 提供修复后的完整代码
3. 编写 Jest 单元测试（覆盖率>85%，含边界情况）
4. 为每个 public 函数添加 JSDoc 注释

[粘贴代码]
```

## 四、批量任务推荐清单

额度充裕时（如非高峰时段），优先处理这些**高价值批量任务**：

```
⚡ 全项目 TypeScript 类型补全
⚡ 为无测试覆盖的模块批量生成单元测试
⚡ 全局 JSDoc / TSDoc 注释生成
⚡ 依赖版本升级 + 兼容性修复
⚡ 将旧 callback 代码批量重构为 async/await
⚡ 生成 OpenAPI / Swagger 文档
⚡ 性能分析 + 优化建议实施
⚡ 代码安全漏洞扫描 + 修复
⚡ 国际化（i18n）字符串提取
⚡ 数据库 migration 脚本生成
```

## 六、Claude Code 并行策略

### Worktree 多任务

```bash
# 同时处理 3 个独立任务
claude --worktree feature/user-auth    # 终端1
claude --worktree fix/payment-bug      # 终端2
claude --worktree test/api-coverage    # 终端3
```

每个 worktree 独立运行，互不干扰。

### 长任务 + 通知

配置完成通知，长任务结束时提醒你：

```json
{
  "hooks": {
    "Notification": [{
      "hooks": [{
        "type": "command",
        "command": "powershell -Command \"Add-Type -AssemblyName System.Windows.Forms; [System.Windows.Forms.MessageBox]::Show('Claude Code 任务完成')\""
      }]
    }]
  }
}
```
